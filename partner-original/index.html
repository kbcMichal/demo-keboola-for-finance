Account Type Mapping (MAP_ACCOUNT_TYPE)

ERP-agnostic. Maps each ERP's account types to 5 standard categories. Tailored per accounting system.

A — Assets (AcctRec, Bank, FixedAsset, OthCurrAsset, DeferExpense, UnbilledRec, OthAsset)
P — Liabilities (OthCurrLiab, AcctPay, LongTermLiab, DeferRevenue, Equity)
R — Revenue (Income, OthIncome)
C — Costs (COGS, Expense, OthExpense)
X — Other (Stat, NonPosting)

Tailored per ERP: Each accounting system has its own account type codes. The mapping table translates them to the 5 standard categories (A/P/R/C/X). This is what makes the pipeline ERP-agnostic.

Both tables can be self-served — via Google Sheets, a Data App, or direct upload. Finance updates mappings without touching SQL.

Self-Service Data Quality

Finance Fixes Issues. Not IT.

Automated validation checks run after every pipeline execution. Finance teams see issues in the app and can resolve them without going back to the data team.

≡

BS Balance Validation

Assets + Liabilities + Equity = 0 per entity per month. Catches FX gaps, missing opening balances, incomplete synthetic entries.

⚠

Unassigned Accounts

GL entries referencing accounts not in DIM_COA. Usually means the CoA mapping is incomplete — finance can add the missing mapping directly.

✖

FININ Assignment Bugs

Journal entries with broken FININ mappings: missing codes, non-leaf targets, unmapped accounts. Pinpoints exactly which accounts need fixing.

⇄

Local vs Consolidated Sums

Verifies that the sum of local entity values matches the consolidated rollup. Catches MAP_FININ_STRUCTURE gaps and sign convention issues.

$

Missing FX Rates

Entity-period combinations with no exchange rate. Prevents silent zeros in converted values.

Thresholds are configurable — defined in the Business Glossary (DIM_METRIC_THRESHOLD). RAG status (Red/Amber/Green) per metric with upper and lower bounds. Client defines what "acceptable" means.
